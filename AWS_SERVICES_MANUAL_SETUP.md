# AWS Services Manual Setup Guide

This guide details exactly how to provision the necessary AWS resources (DynamoDB, S3, SES, Cognito) via the AWS Management Console to support the StudySync backend architecture.

---

## 1. DynamoDB (Database)

Based on `backend_py/shared/dynamodb.py` and the API routes, the backend relies on three separate DynamoDB tables to store users, groups, and sessions. 

**Steps to Create:**
1. Navigate to the **DynamoDB** dashboard in the AWS Console.
2. Click **Create table**.
3. **Table 1:**
   * Table name: `StudySync-Users` *(This matches the default `USERS_TABLE` environment variable)*
   * Partition key: `userId` (Type: **String**)
   * Table settings: Select **Customize settings**, select **On-demand** capacity (to save costs).
   * Click **Create table**.
4. **Table 2:**
   * Table name: `StudySync-Groups` *(Matches `GROUPS_TABLE`)*
   * Partition key: `groupId` (Type: **String**)
   * Table settings: Select **Customize settings**, select **On-demand** capacity.
   * Click **Create table**.
5. **Table 3:**
   * Table name: `StudySync-Sessions` *(Matches `SESSIONS_TABLE`)*
   * Partition key: `sessionId` (Type: **String**)
   * Table settings: Select **Customize settings**, select **On-demand** capacity.
   * Click **Create table**.

## 2. S3 Bucket (File Storage)

The backend (`backend_py/shared/s3.py`) generates pre-signed URLs (`generate_presigned_url`) to allow authenticated users to securely upload and download study materials bypassing the EC2 server to save bandwidth.

**Steps to Create:**
1. Navigate to the **S3** dashboard.
2. Click **Create bucket**.
3. **Bucket name**: `studysync-materials-yourname` *(Bucket names must be globally unique across all AWS users, make sure you configure this name as your `S3_BUCKET` env var)*.
4. **Block Public Access settings**: Ensure block *all* public access is **checked** (Turned ON). The pre-signed URLs generated by your Python code securely bypass this block.
5. Click **Create bucket**.

**Configure CORS:**
Since the React frontend uploads directly to S3 using the pre-signed URL via a PUT request, CORS must be enabled.
1. Click on your newly created bucket.
2. Go to the **Permissions** tab.
3. Scroll down to **Cross-origin resource sharing (CORS)** and click **Edit**.
4. Paste the following JSON:
   ```json
   [
     {
       "AllowedHeaders": ["*"],
       "AllowedMethods": ["GET", "PUT", "POST"],
       "AllowedOrigins": ["*"],
       "ExposeHeaders": []
     }
   ]
   ```
5. Click **Save changes**.

## 3. Simple Notification Service (SNS)

The backend (`backend_py/shared/sns.py`) uses the `sns_client.publish` method to broadcast notification emails to an Amazon SNS Topic when a new session is scheduled via the `/sessions` API endpoint.

**Steps to Create:**
1. Navigate to the **Amazon SNS** dashboard.
2. Click **Topics** in the left sidebar, then click **Create topic**.
3. **Type**: Select **Standard** (FIFO is not required).
4. **Name**: Enter `StudySync-Notifications`.
5. Scroll down to the bottom and click **Create topic**.
6. On the topic's summary page, copy the **Topic ARN** (it looks like `arn:aws:sns:eu-west-1:123456789:StudySync-Notifications`). You will need to save this as the `SNS_TOPIC_ARN` environment variable on your EC2 box.

**Steps to Subscribe:**
Good news! You **do not** need to manually subscribe your users. As part of this application's automated workflow, the moment a brand new user registers in Cognito and logs into the frontend for the first time, the `/users/profile` endpoint automatically creates an AWS SNS subscription for their email!

1. When a new user logs in, they will automatically receive an **AWS Notification - Subscription Confirmation** email in their inbox.
2. The user **MUST** open this email and click the confirmation link.
3. Once confirmed, they will successfully receive all future session scheduling notifications.

## 4. Cognito (Authentication)

Cognito is used by the React frontend to securely register, log in, and retrieve JSON Web Tokens (JWT) which are subsequently validated in the headers by FastAPI.

**Steps to Create:**
1. Navigate to the **Cognito** dashboard.
2. Click **Create user pool**.
3. **Configure sign-in experience**:
   * Select **Email** as the sign-in option.
   * Click **Next**.
4. **Configure security requirements**:
   * Password policy: Default is fine (8 characters).
   * Multi-factor authentication (MFA): Select **No MFA** (for simplicity during development workflows).
   * Click **Next**.
5. **Configure sign-up experience**:
   * Enable self-registration.
   * Required attributes: Add `email` and `name`.
   * Click **Next**.
6. **Configure message delivery**:
   * Select **Send email with Cognito** (AWS provides a small free allowance for this so SES isn't necessary right away).
   * Click **Next**.
7. **Integrate your app**:
   * User pool name: `studysync-pool`
   * Initial app client: Select **Public client**.
   * App client name: `studysync-client`
   * Ensure **Don't generate a client secret** is selected. (React SPA cannot securely store client secrets).
   * Click **Next**, review, and click **Create user pool**.

**Where to use the credentials:**
Once created, click on your user pool and copy the **User pool ID**. Then go to the **App integration** tab and copy the **Client ID** located at the bottom. Inject these into `frontend/src/.env` or `aws-exports.js` so the frontend knows where to authenticate.
